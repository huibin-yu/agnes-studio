## 1. 执行摘要

**Agnes Studio** 是一个中文优先的 AI 图片/视频生成平台，基于 Agnes AI API 构建，面向中文社交媒体创作者和中小企业。

### 当前状态: 功能原型，基础扎实，但尚未达到生产标准

#### ✅ 做得好的地方
- 产品定位清晰：中文创作者 × 用例驱动（小红书封面、电商主图、活动海报）
- 积分账本设计成熟：原子操作 + 幂等退款 + 行级锁
- 端到端流程完整：注册 → 生成 → 画廊 → 作品管理
- API 密钥管理打开了开发者/B2B 通道
- 技术栈现代：Next.js 15 + FastAPI + SQLAlchemy 2.0

#### 🚨 必须解决的严重问题
| 问题 | 严重程度 |
|------|---------|
| **零收入能力** — 充值按钮完全无效 | 🔴 致命 |
| **单一外部 API 依赖**，无熔断/重试/降级（RPM ≤ 20!） | 🔴 致命 |
| **无数据库迁移系统** — Alembic 已引入但从未使用 | 🔴 致命 |
| **积分扣费存在多个竞态条件**，可导致资金泄漏 | 🔴 致命 |
| **SQLite 作为默认生产数据库** — `SELECT FOR UPDATE` 无效 | 🔴 致命 |
| **首页展示虚假统计数据** — "10万+用户"为硬编码 | 🔴 信任危机 |

#### 📊 总体评分

| 维度 | 评分 | 说明 |
|------|------|------|
| 产品完整性 | 6/10 | 核心功能齐全，但缺收入和社区功能 |
| 用户体验 | 5/10 | 流程通但摩擦点多，信任信号缺失 |
| 架构设计 | 6/10 | 基础模式正确，但缺乏生产级基础设施 |
| 后端质量 | 6/10 | 积分系统优秀，但有竞态条件和安全漏洞 |
| 前端质量 | 5/10 | 能用但代码重复严重，状态管理有缺陷 |
| 测试覆盖 | 4/10 | 核心资金路径已测，50%+ 代码无测试 |
| 生产就绪度 | 3/10 | 缺迁移、缺监控、缺备份、SQLite 无法并发 |
| 可扩展性 | 2/10 | SQLite + 本地文件 + 单服务器 ≈ 上限约 10 用户 |

#### 🎯 结论
该产品处于**原型到生产的拐点**。基础是扎实的，但 4 个 P0 Bug、8 个 P1 架构缺口和零变现能力必须在任何真实用户流量之前解决。

## 2. 产品评估

### 解决什么问题
中文内容创作者需要为小红书、微信公众号、电商制作专业图片/视频，但缺乏设计技能和英文提示词能力。

### 目标用户
- 小红书博主 / 店铺运营
- 微信公众号内容编辑
- 电商卖家（产品主图、详情页）
- 市场营销人员（活动海报、社交媒体素材）

### 用户核心任务
1. 生成社交媒体封面图（不需要设计师）
2. 为电商商品列表创建产品展示图
3. 为社交媒体内容生成短视频
4. 从画廊获取灵感、学习他人作品
5. 通过 API 密钥编程调用生成能力

### 竞争定位

| 维度 | Agnes Studio | Midjourney | DALL-E | 可灵/Pika |
|------|-------------|------------|--------|----------|
| 中文原生体验 | ✅ 强 | ❌ 仅英文 | ❌ 仅英文 | ✅ 强 |
| 用例预设 | ✅ 小红书/电商/海报 | ❌ 通用 | ❌ 通用 | ❌ 通用 |
| 图片+视频一体 | ✅ 都支持 | ❌ 仅图片 | ❌ 仅图片 | ✅ 都支持 |
| API 访问 | ✅ 内置 | ❌ 无 | ✅ 有 | ❌ 有限 |
| 生成质量 | ⚠️ 取决于 Agnes AI | ✅ 业界领先 | ✅ 强 | ✅ 强 |
| 社区生态 | ⚠️ 基础画廊 | ✅ Discord 社区 | ❌ 极少 | ⚠️ 基础 |

**核心价值主张**: "不用写英文提示词，不用会设计——用例驱动的中文 AI 创作平台。"

### 关键发现: 隐藏功能
后端已完整实现 **图生视频**（`mode="i2v"`），但前端**零 UI 入口**。这是一个免费的功能扩展机会——后端代码已完全就绪。

### 功能差距分析

| 功能 | 竞品现状 | Agnes Studio |
|------|---------|-------------|
| 图生视频 | Pika/Runway/可灵均已支持 | 后端已实现，前端未暴露 |
| 局部重绘/扩图 | 所有主流平台均支持 | 完全缺失 |
| 图片超分辨率 | Midjourney/ComfyUI 标配 | 未实现 |
| 图片变体生成 | Midjourney "Vary" 核心功能 | 未实现 |
| 批量生成 | Midjourney 每次生成 4 张 | 每次仅 1 张 |
| 种子值控制 | 所有平台均支持 | 数据模型已存，前端无入口 |
| 风格参考/迁移 | Midjourney --sref | 不可用 |
| 画廊详情页 | 所有平台均支持 | 页面不存在（404） |
| 评论/关注 | 基础社区功能 | 仅有点赞和浏览 |

### 变现评估

**当前状态: 完全不可用**

充值页面列出 4 个套餐（10 免费 / 50 币 ¥9.9 / 200 币 ¥29.9 / 1000 币 ¥99.9），但：
- 所有购买按钮**无 onClick 处理器**
- **无支付网关集成**（无支付宝/微信支付/Stripe）
- 充值页面**不在导航栏中**，用户根本找不到

**单位经济学分析**（基于 Agnes AI 当前免费定价）:
- 每张图片售价: ¥9.9/50 = ¥0.198/张
- 每 10 秒视频售价: 20 币 × ¥0.198 = ¥3.96
- 平台成本: ¥0（Agnes AI 目前免费）
- ⚠️ 一旦 Agnes AI 开始收费，利润率将急剧压缩

## 4. 架构评估

### 整体架构

```
┌──────────────┐     ┌───────────────┐     ┌──────────────┐
│  Next.js 15  │────→│   FastAPI      │────→│  Agnes AI    │
│  前端         │     │   后端         │     │  外部 API    │
│  Port 3000   │     │   Port 8000    │     │  RPM ≤ 20    │
└──────────────┘     └───────┬───────┘     └──────────────┘
                             │
                      ┌──────┴───────┐
                      │   SQLite     │  ← 单写锁，无法并发
                      │  (默认)      │  ← FOR UPDATE 无效
                      └──────────────┘
```

### 架构优势
- ✅ 积分账本模式：调用方控制事务边界，原子操作 + 幂等退款
- ✅ JWT Token 版本控制：改密码即时失效所有旧 Token
- ✅ SECRET_KEY 启动校验：防止使用默认密钥部署
- ✅ 全局异常处理器：服务端记录详情，客户端只收到通用消息
- ✅ 视频 URL 提取：防御性解析上游 6+ 种格式变体

### 架构问题

| 优先级 | 问题 | 位置 | 说明 |
|--------|------|------|------|
| 🔴 P0 | **无数据库迁移系统** | `main.py:41` | Alembic 已引入但从未初始化。`create_all` 无法修改已有表 |
| 🔴 P0 | **SQLite 作为生产默认** | `config.py:14` | 单写锁、无行级锁、无法多 Worker |
| 🔴 P0 | **Token 存储在 localStorage** | `stores/auth.ts` | XSS 漏洞可窃取 30 天有效期的 Refresh Token |
| 🔴 P0 | **健康检查使用原始 SQL 字符串** | `main.py:106` | 应使用 `text("SELECT 1")` |
| 🟡 P1 | **重复的限流器实例** | 多个 API 模块 | 每个模块独立创建 `Limiter()`，跨模块限流可能失效 |
| 🟡 P1 | **测试代码出现在生产路径** | `api/videos.py:27-28` | `if video.__class__.__module__.startswith("unittest.mock")` |
| 🟡 P1 | **httpx 客户端无生命周期管理** | `agnes_ai.py:52-59` | 模块导入时创建，从不关闭，连接泄漏 |
| 🟡 P1 | **全局单例服务** | 所有 service 文件 | 无法通过依赖注入替换，测试困难 |
| 🟡 P1 | **Docker 容器以 root 运行** | `backend/Dockerfile` | 无 `USER` 指令 |
| 🟡 P2 | **大量代码重复** | `image_service.py` | `generate()` 和 `generate_img2img()` 约 80% 代码相同 |
| 🟡 P2 | **删除操作绕过 Service 层** | `api/images.py`, `api/videos.py` | 路由中直接执行原始查询 |

### 可扩展性瓶颈

| 规模 | 瓶颈 | 影响 |
|------|------|------|
| 10 倍 | SQLite 单写锁 | 所有写入串行化 |
| 10 倍 | Agnes AI RPM ≤ 20 | 最多 20 个并发生成 |
| 100 倍 | 本地文件存储 | 磁盘耗尽，无 CDN |
| 100 倍 | 客户端驱动的视频轮询 | API 流量爆炸 |
| 1000 倍 | 单服务器架构 | 无法水平扩展 |

### Docker 配置问题
- **SQLite 卷路径不匹配**: Dockerfile 创建 `/app/data`，但 SQLite URL 使用 `./agnes_studio.db`（相对于工作目录 `/app`），卷挂载 `db_data:/app/data` 实际不会持久化数据库文件
- **无环境分离**: 同一个 `docker-compose.yml` 用于开发和生产
- **无资源限制**: 无 `mem_limit`、`cpus` 配置
- **无 TLS 终结**: 两个服务都暴露明文 HTTP

## 5. 后端评估

### 发现的 Bug

| # | Bug | 严重程度 | 文件 |
|---|-----|---------|------|
| 1 | **头像 URL 路径不匹配** — 文件以 `{user_id}_{name}` 保存，但 URL 存为 `/uploads/avatars/{name}` | 🔴 高 | `api/users.py:64` |
| 2 | **视频双重退款竞态** — 并发轮询可同时读到 `credits_charged > 0`，然后都执行退款 | 🔴 高 | `video_service.py:219-227` |
| 3 | **孤儿上游任务泄漏** — `InsufficientCreditsError` 发生时 Agnes 任务已创建，无法取消 | 🔴 高 | `video_service.py:149-158` |
| 4 | **画廊点赞数可变为负数** — 并发取消点赞时无下限保护 | 🟡 中 | `gallery_service.py:176` |
| 5 | **失败记录的 `image_url` 存为空字符串** — `nullable=False` 但语义错误 | 🟡 中 | `models/image.py:33` |
| 6 | **`get_my_videos` 丢弃分页总数** | 🟢 低 | `api/videos.py:91-100` |

### 并发问题

- **图片生成 TOCTOU 竞态**: 预检积分 → 调用 Agnes API（耗时数秒） → 扣费。间隙允许超支。
- **视频创建在扣费前提交**: 记录以 "queued" 状态提交后才扣费。进程崩溃 = 免费视频。
- **画廊浏览量返回旧值**: 先查询再自增，返回的值是自增前的。

### 安全漏洞

| # | 漏洞 | 严重程度 | 说明 |
|---|------|---------|------|
| 1 | **画廊 `media_url` 未校验** | 🔴 高 | 可注入 `javascript:` 或内网 URL，构成 XSS/SSRF |
| 2 | **img2img `image_url` 是 SSRF 向量** | 🔴 高 | 用户 URL 直接传给 Agnes AI，可探测云元数据 |
| 3 | **API 密钥明文存储** | 🟡 中 | 数据库泄露 = 所有密钥立即可用 |
| 4 | **Token 存在 localStorage** | 🟡 中 | XSS 可窃取 30 天有效期的 Refresh Token |
| 5 | **限流器仅基于 IP** | 🟡 中 | 反向代理后所有用户共享同一 IP |
| 6 | **无提示词长度限制** | 🟡 中 | 可发送 100KB 的提示词 |
| 7 | **无内容审核** | 🟡 中 | 提示词原样转发给 Agnes AI |

### 代码质量问题
- **异常链丢失**: `agnes_ai.py` 重新抛出通用 `Exception` 而不使用 `raise ... from e`
- **同步文件 I/O 阻塞异步事件循环**: `filepath.write_bytes()` 在 async handler 中使用
- **httpx.AsyncClient 从不关闭**: 导入时创建，无生命周期管理
- **80% 代码重复**: `image_service.generate()` 与 `generate_img2img()` 几乎相同
- **无重试逻辑**: Agnes API 的瞬态失败（429/503）被视为永久失败
- **无熔断器**: Agnes API 宕机时，每个请求等待 360 秒才超时

| 重复模式 | 出现次数 | 建议 |

## 附录: 跨团队交叉审查

### 发现的冲突与调和

1. **产品说"快速上线功能" vs 架构说"先修基础"**
   → 调和: 先用 1-3 天修紧急 Bug（阶段 0），再用 2 周修基础（阶段 1），然后再加功能。在脆弱基础上加功能 = 更多债务。

2. **增长说"做订阅模式" vs 产品说"做积分包"**
   → 调和: 都做。积分包给轻度用户，订阅给重度用户。订阅是收入支柱。

3. **后端发现"头像 URL 不匹配"但 UX 没发现**
   → 原因: 开发环境下看起来正常（文件已保存，只是 URL 错了），生产环境会静默失败。

### 盲点

1. **无人提及隐私合规** — 平台收集邮箱、用户名和生成内容。面向中国市场需遵守《个人信息保护法》(PIPL)。

2. **无人直接面对 RPM ≤ 20 瓶颈** — 这是房间里的大象。整个平台无法服务超过约 20 个并发生成。必须与 Agnes AI 直接沟通或添加替代供应商。

3. **无人提及比赛截止日期** — 这是为 DreamSeed 比赛构建的。部分速赢项目（暴露图生视频、修复画廊、推荐 UI）应为比赛影响力优先推进。
